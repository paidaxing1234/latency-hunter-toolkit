使用 **latency-audit** skill，对 $ARGUMENTS（未指定则为当前项目里的 C++ 低延迟 / 高频交易引擎热路径代码）做一次完整体检：

- 先定位热路径（tick 回调 → 信号 → 下单、撮合、序列化、无锁队列）
- 按「① 并发 / 锁 / 伪共享 ② 内存 / 缓存 / 分配 ③ syscall / IO / 序列化 / 网络 / 分支」三大类逐项审查
- 每条定位到 `文件:行`，基于真实代码、不臆测
- 按严重度输出一张「热路径体检报告」，并建议用 `perf` / `clang-tidy` 实测验证（**不承诺具体微秒数**）
