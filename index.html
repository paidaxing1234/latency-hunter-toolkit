What it is

Latency Hunter Toolkit is an engineering microscope for quant trading code. You hand it a backtest, a hot path, a connector, market data, or a risk config, and it returns a severity-ranked report of the traps that make backtests look rich and live trading lose money. It does not give you alpha — it makes sure the alpha you think you have isn't a bug.

Five skills

Each skill encodes a hardened review or scaffolding protocol so you don't re-learn the same trap twice.

backtest-guard

Backtest 照妖镜

Catches look-ahead bias, data leakage, overfitting and bogus return accounting that make backtests lie.

latency-audit

热路径猎手

Hunts allocations, locks, syscalls and copies on the C++ low-latency hot path of your trading engine.

connector-forge

连接器锻造

Generates and audits exchange connectors with auto-reconnect, heartbeat watchdog, rate limiting and env-only keys.

orderbook-sanity

数据照妖镜

Validates local orderbook construction, sequence gaps, snapshot resync and market-data integrity.

risk-config-lint

风控体检

Lints risk configuration for missing limits, unsafe defaults and gaps that let a bad config blow up an account.

One-line install

Run this inside Claude Code to register the marketplace, then install the toolkit.

> /plugin marketplace add paidaxing1234/latency-hunter-toolkit

All five skills become available immediately. Every skill prompt is plain text in the repo — read it before you run it.

The "four audits, one forge" loop

A closed loop from idea to live: four review gates plus one generator, so each layer is hardened before the next.

  1. 1 · Backtest audit — prove the edge is real, not a look-ahead artifact (backtest-guard).
  2. 2 · Latency audit — make sure the C++ hot path is fast enough to capture that edge (latency-audit).
  3. 3 · Orderbook audit — confirm the market data feeding both is clean and gap-free (orderbook-sanity).
  4. 4 · Risk audit — verify the risk config can survive a bad day (risk-config-lint).
  5. 5 · Connector forge — scaffold the reliable exchange plumbing that ties it all together (connector-forge).

FAQ

What is Latency Hunter Toolkit?
A free Claude Code plugin that audits and scaffolds quant trading engineering across five skills: backtest credibility, C++ low-latency hot paths, exchange connector reliability, orderbook data sanity, and risk config — for venues such as Binance and OKX.
How do I install it in Claude Code?
Inside Claude Code, run /plugin marketplace add paidaxing1234/latency-hunter-toolkit, then install the toolkit. The five skills become available right away.
Does it find alpha, predict returns, or need my API keys?
No. It is an engineering-review tool only. It generates no signals or alpha, predicts no returns, and never reads or stores API keys — connector scaffolds inject credentials from environment variables.
Which exchanges and languages does it cover?
The connector and orderbook skills target crypto venues such as Binance and OKX over WebSocket/REST. Backtest and risk audits are language-agnostic with a Python focus; latency-audit targets C++ low-latency code.
Is it free and open source?
Yes — free to use, with all source in the public GitHub repository so you can read every skill prompt before running it.
